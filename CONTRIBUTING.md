# 🤝 기여 가이드라인 (Contributing Guidelines)

AWS Cleaner 프로젝트에 관심을 가져주셔서 감사합니다! 여러분의 기여는 프로젝트를 더욱 발전시키는 큰 힘이 됩니다. 아래의 가이드라인을 따라 기여해 주시면 감사하겠습니다.

---

## 🐛 버그 신고 및 기능 제안

버그를 발견하셨거나 새로운 기능을 제안하고 싶으신 경우, 직접 코드를 수정하기 전에 **[GitHub Issues](https://github.com/hslcrb/missile_to_AWS/issues)**를 활용해 주세요.

### 1. 버그 신고 (Bug Reports)
- 이슈를 생성할 때 버그의 증상, 재현 방법, 그리고 사용 중인 환경(Windows 버전 등)을 상세히 기술해 주세요.
- 가능하다면 스크린샷이나 에러 로그를 첨부해 주시면 문제 해결에 큰 도움이 됩니다.

### 2. 기능 제안 (Feature Requests)
- 제안하려는 기능이 왜 필요한지, 어떤 방식으로 동작하면 좋을지 설명해 주세요.
- 프로젝트의 방향성과 일치하는 논의를 거친 후 구현을 시작하는 것이 좋습니다.

---

## 🛠️ 개발 및 풀 리퀘스트 (Pull Requests)

코드를 직접 수정하여 기여하고 싶으신 경우 아래 절차를 따라주세요.

### 1. 브랜치 전략
- 새로운 기능 개발이나 버그 수정은 별도의 브랜치(예: `feature/amazing-feature` 또는 `fix/bug-name`)에서 작업해 주세요.
- 작업이 완료되면 `next-gamma` 브랜치를 대상으로 Pull Request를 생성해 주세요.

### 2. 코딩 스타일 및 규칙
- 본 프로젝트는 **순수 Win32 API 및 C++**를 기반으로 합니다. 가급적 외부 라이브러리 의존성을 추가하지 않는 방향을 지향합니다.
- 변수 및 함수 명명 규칙은 기존 코드(`camelCase` 또는 `PascalCase`)를 존중해 주세요.
- 주석은 가독성을 위해 한글 또는 영어를 선택하여 작성해 주세요.

### 3. PR 제출 전 체크리스트
- [ ] `build.bat`을 통해 빌드가 정상적으로 완료되는지 확인했는가?
- [ ] 기존 기능에 영향을 주지 않는지 충분히 테스트했는가?
- [ ] 커밋 메시지는 변경 사항을 명확히 설명하고 있는가?
