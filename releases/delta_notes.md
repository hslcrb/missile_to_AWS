# 🚀 Delta Release (2026-03-26)

이번 **Delta 릴리즈**는 AWS Cleaner 프로젝트의 **UI/UX 편의성 향상**과 **내부 소스코드 구조 및 디렉토리 최적화**에 중점을 두었습니다.

## ✨ 주요 기능 및 UI 개선 (Features & UI)
- **진행률 및 스피너 UI 도입**: 긴 삭제 작업 중 터미널 로그를 기반으로 현재 진행 상태를 시각적인 애니메이션 스피너(`⠋⠙⠹...`)와 Win32 `ProgressBar`로 표시하도록 개선했습니다.
- **멀티 패스(Multi-pass) 삭제 로직**: 의존성 에러(`DependencyViolation`) 발생 시 애플리케이션이 자동으로 로그를 감지하고, 최대 2회차의 `aws-nuke` 재시도를 통해 누락 자원을 자동 정리합니다.
- **지연 삭제 알림 팝업**: KMS, CloudFront, S3 등 AWS 상에서 삭제 완료까지 오랜 시간(비동기)이 소요되는 특정 자원 그룹에 대해 사용자 교육용 경고 알림 팝업을 추가했습니다.

## 🛠 코드 구조 최적화 및 리팩토링 (Refactoring)
- **main.cpp 코드 최적화 (~17% 중복 제거 효과)**:
  - 잦은 파라미터 반복이 발생하던 `CreateFont()` 로직을 `MakeFont()` 인라인 팩토리 함수 하나로 단일화했습니다.
  - 불필요하게 쪼개져 있던 `WM_TIMER` 분기문 등의 자잘한 빈 블록을 클린업했습니다.
  - 중복 `#include` 문과 미사용 컴포지션을 깔끔히 정리했습니다.
- **디렉토리 재배치 및 `/src` 시스템화**:
  - `main.cpp`, `resources.rc`, 헤더 파일 등 흩어져 있던 핵심 C++ 소스 코드들을 `cpp_doom/src/` 디렉토리 하위로 격리시켰습니다.
  - 단일화된 빌드를 위해 `build.bat`의 참조 경로들을 업데이트했습니다.
  - 각종 개발 도우미 파이썬 스크립트를 `tools/`로 분리하고, 불필요한 임시 파일들을 전부 폐기하여 클린 리포지토리를 달성했습니다.
