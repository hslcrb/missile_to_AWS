# 🚀 AWS Cleaner (formerly Missile-to-AWS)

[![Language](https://img.shields.io/badge/언어-C%2B%2B-blue.svg)](https://en.wikipedia.org/wiki/C%2B%2B)
[![GUI](https://img.shields.io/badge/GUI-Win32_API-lightgrey.svg)](https://docs.microsoft.com/en-us/windows/win32/)
[![Security](https://img.shields.io/badge/보안-Process_Hollowing-red.svg)](#)

**AWS Cleaner**는 `aws-nuke` 도구를 보다 안전하고 직관적으로 실행하기 위해 **순수 Win32 API로 밑바닥부터 재구축한 전용 래퍼(Wrapper) 프로그램**입니다. 파괴적인 AWS 리소스 삭제 작업을 실수 없이 시각적으로 제어할 수 있으며, **인메모리 실행(In-Memory Execution)** 및 **프로세스 할로잉(Process Hollowing)** 기술을 적용하여 보안성을 극대화했습니다.

---

## ✨ 핵심 기능 (Key Features)

### 🛡️ 1. 완벽한 스텔스 실행 및 보안 (Process Hollowing)
*   **인메모리 바이너리 로딩**: `data/` 디렉토리에 XOR 암호화되어 여러 개의 `.dat` 조각으로 분할 저장된 `aws-nuke.exe` 페이로드를 런타임에 메모리상에서만 재조립합니다.
*   **디스크 흔적(Zero-footprint) 제거**: 재조립된 바이너리를 하드디스크에 쓰지 않고, 일시 중지된 시스템 프로세스(`svchost.exe`) 내부에 직접 주입(Injection)하여 실행합니다.
*   **SHA-256 무결성 검증**: 실행 전 모든 페이로드 조각의 해시값을 Windows BCrypt API로 고속 검증하여, 파일 위변조 시 즉각 실행을 차단합니다.

### 🔍 2. 강력한 리소스 필터링 및 한글화 시스템
*   **400+ 리소스 완벽 한글화**: AWS의 방대한 리소스 타입들을 한글로 번역하여 제공합니다.
*   **구글급(Google-grade) 퍼지 검색 엔진**: 영어, 한글, 태그(예: `#컴퓨팅`, `#보안`) 등 다중 키워드를 지원하며, 초성 검색 및 우선순위 알고리즘(정확도 > 접두사 > 태그)을 통해 원하는 리소스를 즉시 찾아냅니다.
*   **즐겨찾기 시스템**: 자주 삭제하는 리소스는 우클릭하여 '즐겨찾기'에 등록할 수 있으며, 드롭다운 최상단에 노란색으로 하이라이트 표시됩니다.

### 🎨 3. 유려하고 직관적인 네이티브 GUI (Win32)
*   일렉트론이나 타우리 같은 무거운 프레임워크 없이 C++, Win32 API, GDI+만으로 구현되어 매우 가볍고 빠릅니다.
*   **인터랙티브 UI 요소**:
    *   마우스 오버, 클릭 시 반응하는 **커스텀 이미지 버튼** (투명도 조절, 색상 반전 등 시각적 피드백 제공).
    *   최상단 로고 지원: 툴팁, 웹 브라우저 연동 및 우클릭 링크 복사 기능.
    *   부드러운 파스텔 옐로우 색상의 세련된 리전 선택 그리드 뷰.
*   **안전 장치 (Nuke Countdown)**: '삭제' 뷰튼 클릭 시 5초간의 타이머가 작동하며, 대기 시간 동안 언제든 '취소' 버튼을 눌러 작업을 롤백할 수 있습니다.

### 💻 4. 실시간 터미널 에뮬레이터 내장
*   백그라운드에서 실행되는 `aws-nuke`의 `stdout`/`stderr` 출력을 GUI 내장 터미널에 실시간 파이핑(Piping)하여 띄워줍니다.
*   `stdin` 리다이렉션을 통해 터미널 내에서 직접 `aws-nuke`의 인증 스크립트(Alias 확인 등) 타이핑이 가능합니다.
*   읽기 전용 보호 및 퍼시스턴트 프롬프트(`>_`) 기능을 지원하여 안정적인 터미널 환경을 경험할 수 있습니다.
*   **IME & Caps Lock 상태 표시기**: 사용자의 현재 키보드 입력 상태(한글/영문, 대문자 여부)를 터미널 상단에 컬러 인디케이터로 실시간 출력합니다.

## 🏗 시스템 아키텍처 (Architecture)

### 프로세스 주입 파이프라인 (Process Injection Pipeline)
1.  **생성**: `svchost.exe`를 `CREATE_SUSPENDED` (일시 중지) 상태로 백그라운드 생성합니다.
2.  **할로잉**: 대상 프로세스의 원래 메모리를 언맵(Unmap)하고 페이로드의 NT 헤더에 맞춰 새로운 공간을 할당합니다.
3.  **매핑**: RAM에서 복호화 및 검증이 완료된 페이로드의 헤더와 섹션들을 원격 프로세스 메모리에 직접 덮어씁니다.
4.  **파이프 연동**: 익명 파이프(Anonymous Pipe)를 생성 및 상속하여 원격 프로세스의 표준 입출력을 MTA GUI의 텍스트 박스와 브릿징합니다.
5.  **재개**: 쓰레드 컨텍스트(ImageBase, EntryPoint)를 수정 후 실행을 재개(ResumeTask)합니다.

## 🚀 빌드 및 실행 가이드

### 요구 사항
*   **Visual Studio 2022** (C++ 데스크톱 개발 워크로드 설치 완료)
*   **Windows SDK**

### 설치 및 빌드
1.  **VS 2022용 x64 Native Tools Command Prompt**를 엽니다.
2.  이 프로젝트의 `cpp_doom` 디렉토리로 이동합니다.
3.  아래의 빌드 스크립트를 실행합니다:
    ```cmd
    build.bat
    ```
4.  빌드가 완료되면 `AWSCleaner.exe` (또는 `missile-to-aws.exe`)가 생성됩니다.

## 📖 상세 사용 방법 (How to Use)

AWS Cleaner는 직관적인 GUI를 제공하지만, 강력한 파괴적 권한을 다루므로 아래의 상세 가이드를 숙지해야 합니다.

### 1단계: 인증 정보 (Credentials) 세팅
*   최상단의 텍스트 박스에 타겟 AWS 계정의 **Account ID**, **AWS Access Key**, **AWS Secret Key**를 순서대로 입력합니다.
*   **보안 장치**: `Secret Key`는 어깨너머 탈취(Shoulder-surfing)를 방지하기 위해 기본적으로 무조건 동그라미(`●`)로 마스킹 처리됩니다. 우측의 **[마스킹 해제 체크박스]**를 활성화하면 입력된 문자를 눈으로 확인할 수 있습니다.

### 2단계: 대상 리전 (Region) 및 리소스 종류 (Resource) 선택
*   **리전 그리드 뷰**: 우측의 파스텔 옐로우 색상 박스에서 리소스들이 존재하는 영역(Region)을 모두 체크하세요. 상단의 **[전체 선택]**을 누르면 가장 빠릅니다.
*   **리소스 필터 드롭다운**: 중앙 좌측의 콤보 박스 클릭 후 타이핑을 시작하세요. 
    *   **초성 및 태그 검색 지원**: 예를 들어 '컴퓨팅'이나 '데이터베이스'로 검색 시 관련 서비스가 나열되며, 영어나 한글 이름 상관 없이 자동 분류됩니다.
    *   **즐겨찾기 추가**: 검색된 드롭다운 항목 위에서 마우스 우클릭을 누르면 **"즐겨찾기에 추가"** 메뉴가 나타납니다. 추가된 항목은 차기 실행 시 언제나 최고 최상단 노란색 배경으로 고정 표시됩니다! (우클릭하여 해제 가능)
    *   `<모두>` 항목을 냅두면 시스템 내 존재하는 전 범위의 리소스를 타겟으로 지정합니다. 매우 주의하세요!

### 3단계: 환경설정 저장 (Save)
*   **[ 저장 ]** 플로피 디스크 아이콘 마킹이 있는 버튼을 클릭하세요.
*   입력하신 모든 정보(계정/키, 체크된 리전 리스트, 선택한 리소스, 즐겨찾기 목록)가 로컬의 `AWSCleaner_mta.json` 파일 및 `external/config.yaml`에 반영됩니다.

### 4단계: 리소스 파괴 (Nuke) 및 안전 장치
*   **[ 물리적 삭제 ] 버튼(빨간 버튼)**을 클릭하면, 터미널에 `[Cleaner] **경고** 5초 뒤 삭제를 진행합니다.`라는 알림과 함께 카운트다운이 발생합니다.
*   이때 **삭제 버튼은 반투명하게 흐려지며 비활성화**되고 우측에 눈에 확 띄는 **빨간색 [ 취소 ] 버튼**이 등장합니다. 
*   **실수 방지**: 만일 세팅을 잘못했다면, 이 5초 안에 언제든 [ 취소 ] 버튼을 눌러 작업을 원점으로 돌릴 수 있습니다.
*   카운트다운이 0이 되면 백그라운드에서 메모리에 주입된 바이너리가 실제 파괴 명령을 개시하며 터미널에 실시간으로 로그를 토해냅니다.

### 💡 기타 꿀팁
*   **최상단 로고 상호작용**: 상단의 `AWSCleaner` 로고에 마우스를 가져가면 손가락 포인터로 변합니다. 클릭 시 본 도구의 오픈소스 GitHub Repository 창이 새 탭으로 띄워지며, 로고를 우클릭하면 해당 링크를 클립보드로 복사할 수 있습니다!

## 📂 프로젝트 파일 구조
*   `cpp_doom/main.cpp`: 전체 UI, GDI+ 렌더링, 제어 로직 및 프로세스 할로잉 엔진 구현부.
*   `cpp_doom/resource_list.h`: 400여 개의 AWS 리소스 영어/한글 매핑 및 다차원 태그 데이터베이스.
*   `cpp_doom/resources.rc` / `resource.h`: 아이콘, Noto Sans 폰트, 커스텀 이미지 Asset 정의 파일.
*   `data/`: 악성코드 분석 회피를 위해 분할되고 암호화된 `.dat` 바이너리 조각 폴더.
*   `AWSCleaner_mta.json`: (실행 시 자동 생성) 설정값, 삭제 기록 로그, 즐겨찾기 리소스를 영구 보존하는 로컬 DB 파일.
