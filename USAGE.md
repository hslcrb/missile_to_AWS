# 📖 AWS Cleaner 사용 매뉴얼 (User Manual)

본 문서는 AWS Cleaner를 처음 사용하는 분들을 위해 기본적인 설정 방법부터 리소스 삭제 단계까지 상세히 설명합니다.

---

## 🏗️ 1단계: 프로그램 실행 및 기본 설정

1.  **빌드**: `cpp_doom/build.bat`을 실행하여 `AWSCleaner.exe`를 생성합니다.
2.  **실행**: 생성된 `AWSCleaner.exe`를 실행합니다.
3.  **자격 증명 입력**:
    - **Account ID**: 대상 AWS 계정의 12자리 ID를 입력합니다.
    - **Access Key / Secret Key**: IAM에서 생성한 자격 증명을 입력합니다.
    - **보안**: `Secret Key`는 기본적으로 마스킹(`●●●`) 처리됩니다. 필요시 우측 체크박스로 확인할 수 있습니다.

## 🔍 2단계: 대상 리전 및 리소스 검색

1.  **리전 선택**: 우측 리전 그리드에서 삭제를 진행할 모든 영역을 체크합니다. (상단 **[전체 선택]** 권장)
2.  **리소스 필터링**:
    - 중앙 드롭다운 메뉴를 클릭하고 원하는 리소스(예: `EC2`, `S3`, `RDS`)를 입력합니다.
    - **초성 검색 지원**: '컴퓨팅'이나 'ㄱ' 등을 입력하면 관련 리소스가 자동 나열됩니다.
    - **즐겨찾기**: 자주 쓰는 리소스는 우클릭하여 노란색 즐겨찾기에 등록할 수 있습니다.

## 💾 3단계: 설정 저장 및 확인

1.  **저장**: 좌측 하단의 **[ 저장 ]** 버튼(디스켓 아이콘)을 누릅니다.
2.  **파일 상태 확인**: 우측 하단 **[ 설정 ]** 버튼을 눌러 `credentials.json`, `config.yaml`, `_mta.json` 파일이 정상 존재하며 최신 상태인지 확인합니다.

## 🚀 4단계: 리소스 파괴 (Nuke)

1.  **실행**: 빨간색 **[ 물리적 삭제 ]** 버튼을 클릭합니다.
2.  **안전 카운트다운**: 5초간의 대기 시간이 주어집니다. 이 시간 동안 실수를 인지했다면 즉시 **[ 취소 ]** 버튼을 누르세요.
3.  **터미널 확인**: 5초 뒤 실시간 터미널 창에 `aws-nuke` 본체의 실행 로그가 파이핑됩니다.
4.  **최종 확인**: 작업이 완료되면 터미널에 성공 또는 실패 결과가 출력됩니다.
